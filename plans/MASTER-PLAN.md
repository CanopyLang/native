## 1. North star & definition of done

**North star:** lift canopy/native to the most reliable app framework available — *no errors, full compatibility* — where every word is **measured and CI-gated, not asserted.**

This splits into two halves, each with a hard, falsifiable bar.

### 1a. The precise, honest guarantee scope (lead with the asterisks)

"No errors" is a **scoped** claim. The credibility bomb is publishing "cannot crash" and having someone reproduce a crash. So the guarantee leads with what is **NOT** guaranteed, and every external surface (README, marketing, any A2UI/SDUI conformance page) must carry these asterisks or it is wrong.

**Positive guarantees (correctness-by-construction, each backed by a live enforcement file):**

| Guarantee | Enforced by (live file) |
|---|---|
| No `null`/`undefined` in user code; no "undefined is not a function" in compiled output | bundle F7/ReferenceError gate: `scripts/verify-iife-no-f7.js`, `scripts/build-app-bundle.sh` |
| Exhaustive pattern matching; no unhandled case | the pinned compiler (`scripts/compiler-pin.env` → sibling `compiler`) |
| Effects only via `Cmd`/`Sub`/managed runtime (no ambient I/O) | the TEA runtime in `core`; the command seam `harness/run-command.js`, `package/external/native.js` |
| A JS re-entry fault becomes a red-box overlay, not `SIGABRT` | `guardJsCall` in `host/android/app/src/main/jni/CanopyHostJni.cpp:164` → `CanopyRedBox.java` |
| A structurally-bad or hash-mismatched release bundle is refused (fail-closed) | `scripts/check-release-bundle-security.sh` (ci-test step 12) |
| Reload failure recovers to last-known-good (dev loop) | DEV-11 path in `package/external/native.js` |

**The asterisks (the ONLY ways through — each must have a test proving it surfaces recoverably, not as a bare abort):**

- **Stack overflow** on unbounded recursion (purity does not bound the stack).
- **Hermes OOM** (allocation is not type-checked away).
- **Ports / FFI boundary faults** — a capability `.so`, a malformed port payload, or a host native module can fault below the JS boundary.
- **Structural `==` on values containing functions** — runtime error, by language design.
- **Host-side C++ / Yoga / NDK faults** — a segfault inside Yoga, the JSI marshalling layer, or a capability native lib still aborts the process *today*, below `guardJsCall`. **This is the single biggest hole** and is REL-2.

### 1b. Definition of Done (the measurable bar — adopted from the adversarial critic, sharpened)

A release is "most reliable / full compatibility" only when ALL of these are simultaneously green:

1. **Guarantee scope** — `docs/guarantee.md` exists, leads with the asterisks, cites a live enforcement file per positive claim, and a CI doc-lint fails if any cited gate file is deleted. No external surface uses "no errors" / "cannot crash" without the asterisks.
2. **Host-crash floor** — a deliberately-injected `SIGSEGV`/abort at the JSI/Yoga boundary surfaces a red-box (recoverable) or a clean captured-trace fast-fail (unrecoverable) on **both** Android (instrumented) and iOS (XCUITest) — never a bare abort — proven by a required test on each.
3. **Crash-free metric** — ONE crash-free-session metric, real denominator (sessions from ≥1 shipped store build on physical devices, not emulator), per-platform + per-buildId, published with its explicit session count; a CI gate fails below a committed floor (start 99.0%, ratchet). Pre-shipping numbers are labeled emulator-only.
4. **Shipped** — Lumen installable from a Play internal track AND a TestFlight internal group; the full `pick → restore → before/after → paywall → save → share` flow passes on a physical arm64 Android device and a physical iPhone. Signed AAB + signed `.ipa` produced by CI with **real upload keys** (not the dev keystore).
5. **Real-device perf** — `harness/perf-ledger.json` has arm64 canopy rows (`verified:true, device:true`) for Android+iOS, a verified RN 0.76.9 reference on the **same device/ABI** (`rn.verified=true`), `bundleKind=="hbc"`, and all 5 numeric gates + `coldTtiMultiple` are **blocking** with an abi-equality guard rejecting cross-ABI verdicts. The emulator `counter=62% / depth30=75%` jank is explained as emulator artifact (committed arm64 delta) or fixed under the 5% floor on arm64.
6. **Autolink / compatibility** — a third-party stranger package boots on-device with a git-clean `host/` and `package/` tree and zero native source files in the app (zero-edit gate, required/green); `docs/compatibility-matrix.json` vs an Expo SDK baseline is CI-gated; conflicting-version resolution dedups-or-fails-loud deterministically.
7. **Cross-platform parity** — marshalling + capability + layout (incl. RTL) parity driven by single-sourced, byte-identical vector corpora asserted by **both** hosts, each value reproduced by an independent on-Linux oracle; a deliberate one-host divergence reds a device-free gate.
8. **Reproducible build** — two clean builds at the pinned compiler from the same SHA produce byte-identical bundle + `.hbc` + manifest buildId, enforced as a required CI job.
9. **OTA trust** — manifest carries an ed25519 signature; the host refuses to eval any OTA bundle whose `sha256 != buildId`, whose signature/key fails, or whose `runtimeVersion` mismatches, and auto-reverts to embedded last-known-good on a fatal within the watchdog window. OTA scope documented bug-fix-only (App Store 2.5.2).
10. **Supply-chain** — a dependency-CVE scan gate (vendored Hermes/Yoga/fbjni/NDK + any Maven/Pod pulled by interop) runs in CI, an SBOM is published, `SECURITY.md` states a CVE patch-response process and signing-key rotation plan.
11. **Flake control** — every required device/emulator/Appium gate has a documented retry/quarantine policy and a tracked flake rate.
12. **Governance/sustainability** — `GOVERNANCE.md`, `CONTRIBUTING.md`, `SECURITY.md`, `FUNDING.md`, `rfcs/` template committed and linked from README; bus-factor-1 honestly stated with a reproducible build-from-pin + secret/SHA succession plan.

## 2. Strategy in 5 bullets

1. **Stand on the three defensible #1 hills and make them measured, not claimed.** (a) *Correctness-by-construction* — convert the scattered "no errors" comments into one honest, CI-enforced `docs/guarantee.md` and close the one real hole (host C++ crash floor). (b) *Zero-churn upgrades* — no Fabric/TurboModules means insulation from RN's New-Arch treadmill; the ABI pin gates (`scripts/check-abi.sh`, `check-vendor-pins.sh`) already enforce this — keep them, add CVE scanning. (c) *One pure-typed language drives both platforms* — prove it with the cross-platform parity vector contract (REL-6) and a third host (web, REACH-5).
2. **Weaponize the compiler as the team-size multiplier.** A solo dev with a niche zero-training-data language buys leverage by (a) shipping a canopy/native `llms.txt` + compile-gated corpus so plain assistants are usable *today* (AAG-1), and (b) exposing `canopy check --report=json` through an MCP verify→repair loop (AAG-2). The compiler is the assistant's reliability oracle. Defer the speculative SDUI/A2UI research bets until the reliability+shipping core is green.
3. **Ship-and-measure BEFORE marketing.** The headline "crash-free %" and "competitive perf" are meaningless until one app ships to a store on real arm64 hardware, producing a real denominator and a real RN head-to-head on the same device. Until then, every number is labeled emulator-only. No external "most reliable" claim ships before `docs/guarantee.md` (REL-1) is in place.
4. **Every milestone ships its device-free half FIRST.** Make it a structural rule: the source-shape guard, the vector gate, the headless-Hermes assertion, the zero-edit-git-clean check are solo-doable *now* and lock a proven bar even if the on-device/account half stalls on funding. A funding stall then degrades a milestone from "proven on device" to "proven structurally" — never to "unproven."
5. **Be explicit about what one unfunded dev cannot do alone.** Real-device validation, store shipping, device-farm perf, the iOS Mach-exception handler, and the funding/governance survival problem all need an **Apple Developer account ($99/yr), Google Play ($25 + signing keys), Firebase Test Lab / BrowserStack credentials, physical arm64 devices, and ideally a second maintainer.** These are tagged `needs-help-or-funding` and are honestly the gating constraint, not engineering.
